Project Overview: Background Changer

The project is a simple Background Changer application.
It consists of a full-screen element where the background color changes based on which button a user clicks (e.g., Red, Green, Blue, etc.).
The layout is built using Tailwind CSS for easy styling.

Setting Up the Environment

The project is created using Vite, and the folder is named 04bgChanger.
Tailwind CSS is installed as a development dependency.
Essential cleanup is performed: unnecessary CSS files (like App.css) and default "boilerplate" code are removed to start with a clean slate.

Implementing State Management

To make the UI "reactive," the useState hook is used to track the current color.
The state is defined as const [color, setColor] = useState("olive"), where "olive" is the default starting color.
When the color variable changes, React automatically reflects that change in the UI.

Designing the UI with Tailwind

Main Container: A div is created with a width of w-full and a height of h-screen to cover the entire browser window.
Inline Styling: Because the background color is dynamic, it is applied using an inline style object: style={{backgroundColor: color}}.
Navigation Bar: A fixed bar is placed at the bottom of the screen to hold the color buttons.
Styling Buttons: Buttons are styled with rounded corners, padding, and shadows to look professional.

Handling Button Clicks (Important Concept)

A common mistake in React is trying to call a function directly inside onClick, such as onClick={setColor("red")}.
The Problem: onClick expects a function reference, not the result of a function. Writing it with parentheses executes it immediately when the page renders.
The Solution: Use a callback function (arrow function): onClick={() => setColor("red")}. This ensures the function only runs when the button is actually clicked.
