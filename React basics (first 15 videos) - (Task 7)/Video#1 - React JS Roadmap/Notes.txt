Overview of the Series

The series focuses on learning React JS through a project-based approach.
The goal is to provide confidence in coding by building about 8 to 10 projects, ranging from basic to complex.
It emphasizes learning the fundamentals and understanding the "why" behind the technology.

Why Learn React?

High Market Demand: React is currently a very trending library with many job opportunities.
Managing Complexity: It makes it much easier to build and manage complex front-ends where many parts of a page need to update at once.
Note of Caution: You should aim to be a JavaScript Developer or Software Developer rather than just a "React Developer," as technologies change over time.

Prerequisites

Mastery of JavaScript: You should not jump into React with just basic knowledge. You need a strong understanding of execution contexts, functions, and DOM manipulation.
Confidence: You should start React only when you feel confident that you could do most tasks with plain JavaScript but want React to make things faster and easier.

The Origin of React

React was created by Facebook to solve the "Ghost Message" problem.
This problem happened when the UI (the display) and the State (the data) were not in sync, causing notification icons to show unread messages even after they were read.
React was developed to ensure that the JavaScript state and the DOM (UI) stay synchronized.

React: Library vs. Framework

React is a Library: It is compared to a "cool dude" because it offers more freedom and flexibility in how you write code.
Frameworks: Frameworks (like Next.js) are compared to the "military" because they have very strict rules and naming conventions that must be followed.

Main Topics to Learn in React

Core Concepts: This includes understanding State (variables that reflect in the UI), JSX (writing HTML-like code inside JavaScript), and Component Reusability.
Props: Short for properties, these are used to pass data into components.
Hooks: Special functions like useState and useEffect that help manage changes in the application.
Routing: Since React creates Single Page Applications (SPA), you need additional tools like React Router DOM to navigate between different "pages" without reloading the browser.
State Management: For very large apps, tools like Redux Toolkit or the Context API are used to manage data across the whole application.

Advanced Topics and Future Path

BaaS (Backend as a Service): You can use services like Appwrite or Firebase to quickly add backend features like databases and authentication to your React apps.
Frameworks on top of React: After learning React, you can move to Next.js, which solves problems like SEO (Search Engine Optimisation) and browser rendering speeds.
Career Shift: Once you know React and JavaScript, you can also choose to move into Full Stack or Backend development.
