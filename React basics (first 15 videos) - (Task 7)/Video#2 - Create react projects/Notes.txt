Getting Started and Tools Needed

Learning Style: The "learn by doing" approach, starting directly with a project to build confidence.
Platform: Development is platform-independent; you can use Windows, Mac, or Linux.

Essential Tools:

Code Editor: VS Code is recommended, though others like Sublime or Vim work too.
Node.js: Needed to run JavaScript outside the browser. You can check if it's installed by typing node -v in your terminal.
Official Documentation: Always refer to react.dev for the most up-to-date information.

Creating a React Project (Two Methods)

Method 1: Create React App (CRA)
Command: npx create-react-app my-project-name.
Pros/Cons: It is the classic way but is considered "bulky" and slow because it installs many files you might not need.

Method 2: Vite (Recommended)
Command: npm create vite@latest.
Pros/Cons: It is much faster and lighter than CRA. 
After running the command, you select "React" and "JavaScript," then run npm install to set up the dependencies.

Understanding Project Structure

package.json: This is the most important file for understanding a project. It lists the project name, version, and dependencies (like React and React-DOM).
node_modules: A large folder that contains all the code for the libraries your project uses.
Scripts: The package.json file contains scripts to run the project:
In CRA: npm run start.
In Vite: npm run dev.
Build Process: The npm run build command creates a "build" folder with optimized HTML, CSS, and JavaScript. This folder is what is actually sent to users in production.

React Libraries

React Core: The main library that handles the logic of components.
React-DOM: Used specifically for web applications to connect React with the browser.
React Native: Used if you want to build mobile apps instead of websites.

Cleaning Up the Project

Removing Boilerplate: New projects come with many extra files (logos, CSS, testing files) that can be confusing for beginners.
Simplifying Source (src): To see how React truly works, the video demonstrates deleting extra files like App.css, logo.svg, and test files.
Hello World: By cleaning the App.js or App.jsx file and returning a simple <h1> tag (JSX), you can see a clean "Hello World" or "Chai aur React" on the screen.

Execution Flow: React works by injecting your JavaScript code into a single HTML element (usually an ID called "root").
Library vs. Framework: Because React is a library, file naming is flexible, but it's important to understand the flow between the main entry point and your components.
