Importance of DOM Knowledge

Having a strong foundation in the DOM is essential for cracking technical interviews, as many problems require coding DOM manipulations on the spot.
Understanding these basics helps developers quickly grasp advanced concepts in frameworks like React, such as the Virtual DOM and reconciliation.

Navigating Element Relations

The DOM is a tree structure, and understanding the relations between "parents" and "children" is key to adding or removing elements.
Parent to Child: You can access all children of an element using the children property, which returns an HTMLCollection. 
You can also specifically target the firstElementChild or lastElementChild.
Child to Parent: If you have a child element, you can move directly to its parent using the parentElement property.
Siblings: You can move from one element to the next one at the same level using nextElementSibling (e.g., moving from Monday to Tuesday in a list).

Understanding Nodes vs. Elements

In the DOM, everything is a Node, including HTML tags, comments, and even the line breaks (text) between your code.
The childNodes property returns a NodeList that is often much longer than expected because it counts every line break as a "text node".
React often manipulates these child nodes behind the scenes to update the UI efficiently.

Creating New Elements

To create a new tag (like a <div> or <h1>), use the method document.createElement('tagName').
When an element is first created, it exists only in the computer's memory and is not yet visible on the web page.

Adding Properties and Styles

Attributes: You can add a class using .className and an ID using .id.
Custom Attributes: Use setAttribute('key', 'value') to add any attribute, such as a title. 
This is often preferred by developers because it is a more direct way to set values.
Styling: You can apply CSS styles directly using the .style property (e.g., div.style.backgroundColor = "green").

Adding Text Content

There are two main ways to add text to your new element:
innerText: A simple way to set the text, but it involves extra steps for the browser as it "overwrites" content.
createTextNode: A more optimized approach where you create a separate text node and then attach it to your element using appendChild.

Attaching Elements to the Page

To make your memory-stored element appear on the website, you must "attach" it to the document.
A common way to do this is by using document.body.appendChild(yourElement), which places the new element inside the body of the webpage.

Dynamic Element Generation

Once you understand how to create one element, you can put that logic into a function.
By using a loop, you can then call that function multiple times to dynamically generate many elements (like a list of products or cards) with different colors and text.
