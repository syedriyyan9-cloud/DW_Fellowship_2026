Introduction to Iterations: 

Loops are used to repeat a block of code multiple times. In programming, they are also frequently referred to as iterations or iterators.

The Basic for Loop Syntax:
A standard for loop consists of three main parts: initialization (setting a starting variable), condition check (deciding how long to run), and increment/decrement (updating the variable).
Execution Flow: First, the variable is initialized (only once). Then, the condition is checked. If true, the code inside the curly braces runs. Finally, the variable is updated, and the condition is checked again.
Block Scope in Loops: Variables declared inside the for loop (like let i = 0) are restricted to that loop's block scope. They cannot be accessed outside the curly braces, which prevents code "pollution" and bugs.

Nested Loops:

You can put a loop inside another loop (an "inner loop" inside an "outer loop").
How it works: For every single step of the outer loop, the inner loop runs its entire cycle from start to finish.
A common practical example of nested loops is generating multiplication tables.

Looping Through Arrays:

Arrays are commonly traversed using loops by checking the .length property of the array.
It is important to remember that arrays are zero-indexed. If a loop tries to access an index beyond the array's length, JavaScript returns undefined.
The Danger of Infinite Loops: If the condition in a loop is never met (for example, if you forget to increment the variable), the loop will run forever, which can crash your application or consume excessive memory.

Control Keywords: Break and Continue:

break: This keyword immediately stops the entire loop and exits. It is used when you have found what you need or a specific condition is met, and you don't want to process the rest of the loop.
continue: This keyword is like a "skip" or a "pardon." It stops the current iteration for just that one specific step and jumps to the next iteration of the loop without exiting entirely.
