Tools for Deep Exploration

JSVU (JavaScript Engine Version Updater): A tool used to install and manage different JavaScript engines like V8, SpiderMonkey, and JavaScriptCore.
v8-debug (d8): A special debugging version of the V8 engine that allows developers to see internal details like memory allocation and how an array is classified.
Special Flags: Using the flag --allow-natives-syntax allows the use of internal functions like %DebugPrint(array), which reveals the "Kind" or optimization type of an array.

Two Main Categories of Arrays

Continuous (Packed) Arrays: These are arrays where every index has a value. They are highly efficient for the engine to read.
Holey Arrays: These are arrays that contain "holes" (empty spaces) where no value has been assigned. These are significantly less optimized.

The Three Element Types

Inside the categories above, JavaScript further classifies elements into three types:
SMI (Small Integer): The most optimized type, used when the array only contains whole numbers.
Double: Used when the array contains floating-point numbers (decimals) or special values like NaN and Infinity.
Packed Elements: A general type used when the array contains a mix of types, such as strings, objects, or functions.

The Optimization Hierarchy

The engine prioritises arrays based on how easy they are to process. The order from most optimized to least optimized is:
Packed SMI (Best performance)
Packed Double
Packed Elements
Holey SMI
Holey Double
Holey Elements (Worst performance)

The "One-Way" Downgrade Rule

Irreversible: Once an array is "downgraded" to a less optimized type (e.g., adding a string to a number array), it can never be upgraded back to its original state, even if you delete the string.
Triggers: Simply adding NaN or Infinity to an SMI array will permanently turn it into a Double array.

Why Holes are Expensive

Accessing a value in a holey array is slow because the engine must perform multiple expensive checks:
Bound Check: Checks if the index is within the array length.
Property Check: Checks if the index exists as a property of the array object.
Prototype Check: Checks the entire prototype chain to see if the value is defined elsewhere.
hasOwnProperty Check: One of the most expensive checks in JavaScript, performed because arrays are objects internally.

Best Practices for Performance

Avoid Holes: Try not to create empty spaces in your arrays.
Better Initialization: Instead of pre-allocating an array with a specific size (which creates holes), it is often better to start with an empty array [] and push elements into it.
Use Built-in Methods: Prefer using native methods like forEach, map, and for...of. These are heavily optimized by browser engines and are usually faster than custom-built loops.
