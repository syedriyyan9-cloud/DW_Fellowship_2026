Nature of Timing Functions

Functions like setTimeout and setInterval are not actually part of the core JavaScript language.
They are provided by Browser-based APIs (Web APIs) and allow JavaScript to perform tasks outside the main execution thread without blocking other code.

setTimeout and clearTimeout

setTimeout: This function is used to execute a piece of code once after a specific delay (measured in milliseconds).
Syntax: It requires a "handler" (the function to run) and a "timeout" (the delay).
clearTimeout: This is used to cancel a scheduled timeout before it runs. To use it, you must store the reference (ID) of the setTimeout in a variable.
Example: The video shows a heading that changes from "Chai aur Code" to "Best JS Series" after a 2-second delay unless the "Stop" button is clicked.

setInterval and clearInterval

setInterval: Unlike timeout, this function runs code repeatedly at every set interval (e.g., every 1 second).
Parameters: You can pass additional arguments to the function being called through the third parameter of setInterval.
clearInterval: This stops the repeating execution. Like the timeout version, it requires the specific ID of the interval to know which one to stop.
