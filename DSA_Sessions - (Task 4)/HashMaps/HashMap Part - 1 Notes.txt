What is a hashmap?

A hashmap is a highly efficient data structure used to store information in key-value pairs, where every key must be unique. 
It functions by linking a specific key to a specific value, similar to how a phone book allows you to search for a name to instantly 
find a corresponding phone number.
The most significant advantage of a hashmap is its speed; it supports basic operations like searching, inserting, and deleting 
data in constant time (O(1)) on average. 
In different programming languages, these are often referred to as dictionaries (Python) or unordered maps (C++).

What is hashing?

Hashing, which can also be called mapping, is the process of transforming a large, variable-sized input into a smaller, 
fixed-sized output index using mathematical formulas known as hash functions. 
The architecture of hashing generally involves three components: the key (the input being processed), 
the hash function (the formula that computes the index), and the hash table (an array that stores the values at the computed indices).
While hashing is extremely fast, it can encounter an issue called collision, which happens when two different keys generate the same 
index; this is typically resolved using methods like chaining or open addressing.

What is a hash set?

A hash set is a data structure that represents a collection of unique keys only, meaning there are no associated values attached to them.
Its primary purpose is to store a set of items where every element is guaranteed to be distinct, with no duplicates allowed. 
In the Python programming language, these are simply called sets, whereas in C++, they are known as the unordered set data structure.

Need for hash data structure

The need for hash data structures has grown because data is increasing exponentially, making traditional methods like arrays 
inefficient for searching. 
While arrays allow for fast data storage, searching for a specific element in an array takes a significant amount of time (O(logn)), 
which becomes a major problem for massive datasets. 
Hashing bridges this gap by providing a way to store, search, and retrieve data all in constant time (O(1)). 
Additionally, hashing is necessary for handling complex data, such as strings or words, that cannot be easily mapped to simple array
indices, making it a vital tool for performance, scalability, and memory management.

Components of hashing

The hashing architecture is made up of three main components that work together to manage and store data. 
First is the key, which is the input, such as a string or an integer, that is fed into the system. 
Second is the hash function, which is a mathematical formula that receives the key and computes a specific hash index. 
Finally, there is the hash table, which is an array that stores the actual values at the unique indices generated by the hash function.

Key value pair

A hashmap is a data structure that stores information in key-value pairs, ensuring that every key used is unique. 
In this system, you store multiple unique keys, and against each of those keys, there is a specific value associated with it. 
For example, a hashmap can be used to map a word (the key) directly to the number of times it appears in a text (the value). 
This structure is commonly used in programming, such as dictionaries in Python or unordered maps in C++, to link distinct pieces of information together.

Phone book analogy

The phone book analogy is a simple way to understand how a hashmap works in the real world. 
When you need to call a friend, you do not scroll through every single number in your phone one by one; 
instead, you search for a name to find the phone number instantly. 
In this example, the names of your friends act as the unique keys, and their phone numbers are the values linked to those keys. 
This reflects how a hashmap provides instant data retrieval in constant time rather than searching through an entire list.

Advantages of hashing

Hashing provides several significant advantages, the most important being that it allows for storing, searching, and retrieving 
data in constant time (O(1)), which is much faster than searching through a traditional array. 
It is highly efficient for basic operations like insertion and deletion and is ideal for building associative data structures where 
keys are linked to specific values. 
Hashing also helps with memory reduction by allocating manageable space and offers excellent scalability when working with massive 
datasets. 
Furthermore, it is a vital tool for security, specifically in cryptography and verifying the integrity of data.

Collision in hashing

A collision is an edge case in hashing that occurs when two or more distinct keys generate the exact same hash value or index. 
For example, if a hash function simply sums the characters of a string, words like "moon" and "noom" would point to the same index 
because they contain the same letters. 
This is a problem because it disrupts the ability to insert and search for data efficiently, showing that hashing architecture is not always perfect.

Chaining and open addressing

Chaining and open addressing are the two primary techniques used to resolve the issue of collisions. 
Chaining involves pointing a hash table index to another data structure, such as a linked list, where multiple values can be stored 
at the same index address. 
Open addressing resolves collisions by searching for the next possible empty space in the array based on specific conditions and 
adjusting the new data into that available spot instead.

Load factor in hashing

The load factor is a ratio that indicates how full a hash table has become and serves as a decisive parameter for when to resize or rehash the table. 
It is calculated using a simple formula: the total number of elements currently in the hash table divided by the total size of the table. 
A lower load factor is ideal because it means the keys are distributed more uniformly across the table, which significantly 
reduces the probability of a collision occurring.

Ordered map vs unordered map

The main difference between these two structures is how they handle data arrangement and speed. 
An ordered map (known as map in C++) sorts data by the key, which results in a slower lookup time of O(logn). 
An unordered map does not follow any specific order, but it is much more efficient because it allows for a faster lookup time of O(1).
Engineers typically use an ordered map when they specifically need sorted data, while an unordered map is chosen when the primary 
goal is maximum speed and performance.
